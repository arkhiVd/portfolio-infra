Two distinct layers: a global delivery network for the frontend and a regional serverless stack for the backend API.

$ man architecture

The Frontend — delivery layer

Designed for performance, security, and low cost by offloading all traffic to the AWS global edge.

  • Private S3 bucket — all static assets in a secure, non-public bucket.
  • CloudFront CDN — public entry point, global caching, HTTP→HTTPS redirect.
  • Origin Access Control — only this distribution can read the bucket; no direct origin access.
  • ACM & Route 53 — free auto-renewing TLS cert + alias records from the custom domain to the CDN.

The Backend API — application logic

An intelligent unique visitor counter, not a naive hit counter — a real serverless + data-privacy pattern.

  • AWS Lambda (Python) — pseudonymizes visitor IPs with a salted HMAC-SHA256; secret held in an env var.
  • DynamoDB — single-table design: one item holds the aggregate count, others store IP hashes. A fast GetItem check decides if a visitor is new before an atomic UpdateItem avoids race conditions.
  • API Gateway — secure REST endpoint via Lambda proxy integration; CORS scoped to the frontend domain.

The Automation Layer — IaC

The whole project declared in Terraform — no manual console clicking, fully reproducible.

  • Remote state — Terraform state in an S3 backend, the foundation for CI/CD automation.
  • Modular design — split into independent frontend and backend Terraform projects.
  • Real problem-solving — debugged S3 BucketOwnerEnforced vs legacy ACLs and used Terraform's implicit dependency graph to resolve the CloudFront ↔ S3 circular reference.

The Deployment Engine — CI/CD

"Push-to-deploy" GitHub Actions pipelines automating test and deploy for both repos.

  • Secure auth — passwordless OIDC connection to AWS; no long-lived keys stored as secrets.
  • Backend pipeline — Python unit tests + tfsec scan gate the build before terraform apply.
  • Frontend pipeline — applies infra, then uses Terraform outputs to aws s3 sync the site and cloudfront create-invalidation to purge the cache instantly.
