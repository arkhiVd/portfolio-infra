ClearSky — Multi-Account AWS Posture & Cost Platform

A serverless security-posture, cost, and inventory platform for AWS. Deterministic detectors scan every enabled region on a daily schedule, findings live a full lifecycle in DynamoDB, and an agentic AI investigator answers questions against the live account through a read-only tool-use loop — all behind a Cognito-authenticated dashboard on CloudFront.

$ echo $GOAL

Answer two questions for any AWS account, continuously: what is running, and is it safe? ClearSky scans an entire organization — the management account plus any number of member accounts onboarded from the dashboard — with 24+ deterministic, read-only detectors, scores the security posture, tracks cost against baselines, and lets an AI investigator dig into any finding without ever getting write access. The whole platform is serverless and stands up or tears down with one pipeline run.

$ ls highlights/

Deterministic Detection, AI Explanation

24+ detectors are pure functions — API responses in, findings out — fixture-tested and reproducible. AI is additive: it investigates and explains but never decides what's a finding.

Agentic AI Investigator

An LLM runs a tool-use loop against the live account: aws_read allowlisted to describe/list/get calls, findings, and Cost Explorer. Read-only IAM is the hard boundary — the agent physically cannot mutate anything.

Multi-Account by Assume-Role

Member accounts onboard from the dashboard: paste a role ARN, the API validates it by assuming it, and an SSM registry tracks the fleet. Scans fan out cross-account on read-only roles.

Zero-Dependency Lambda Backend

The entire engine — scanner, API, chat, diagram generator, JWT verification (RS256 in pure stdlib) — runs on Python stdlib + boto3. No layers, no packages, nothing to patch.

$ man architecture

The detection engine

Rules detect; AI explains. Detection stays deterministic and testable.

  • Pure-function detectors — each detector takes recorded API responses and returns findings; 113 unit tests run on JSON fixtures, no AWS needed.
  • Finding lifecycle — DynamoDB tracks new / open / resolved per account+region+resource; partial scans reconcile only the accounts they actually scanned.
  • Coverage — cost waste (EBS, EIP, S3 hygiene, idle EC2/RDS, orphaned ELBs, log retention) and posture (MFA, key age, open SGs, public buckets, encryption, CloudTrail), plus a day-over-day posture score.

The AI investigator

An agent with real access to the account — and a hard read-only ceiling.

  • Tool-use loop — the LLM iterates over aws_read (allowlisted to describe_/list_/get_), findings, Cost Explorer, and the architecture model until it can answer; a denylist blocks secrets and data-plane reads.
  • Provider-agnostic — a zero-dependency OpenAI-compatible client (urllib); the API key is entered in the dashboard and stored as an SSM SecureString, never in code or state.
  • Architecture diagrams — a separate worker discovers live resources and renders a deterministic draw.io file with inferred data-flow edges; AI can annotate, never invent.

The security model

Every boundary enforced twice — at the edge and in the code.

  • API reachable only via CloudFront — the Lambda function URL requires OAC SigV4; direct calls fail. The Lambda then re-verifies every Cognito JWT in-process (pure-stdlib RS256).
  • Custom Cognito login — USER_PASSWORD_AUTH against the Cognito API from a hand-built page; no hosted UI, no client secret, admin-only user creation.
  • Read-only scanning — scanner, chat, and diagram roles are read-only everywhere; member accounts grant a scoped clearsky-readonly role whose trust policy is the real gate.

Operations & CI/CD

Fully serverless and pay-per-use; ships and tears down through one pipeline.

  • OIDC pipelines — GitHub Actions with no static keys: read-only plan role on PRs, scoped apply role on main; state in a versioned, TLS-only S3 bucket injected at init time.
  • One-push deploy — a push applies Terraform, bakes config.js from outputs, syncs the dashboard to S3, and invalidates CloudFront; destroy empties every managed bucket first.
  • Pay-per-use everywhere — Lambda, DynamoDB on-demand, CloudFront, Cognito; idle cost is effectively zero with a budget guardrail watching for drift.
